# EduViz - Student Results Dashboard

## Project Overview
A multi-page dashboard displaying exam results for 20 students across 5 subjects. The landing page acts as a global student directory, while each subject has its own dedicated page with rich visualizations. Data is generated by a Python script and served as static JSON/CSV files.

## Objectives
- Generate realistic student data for Mathematics, English, Business Studies, Science, and History.
- Build a spacious, desktop-optimized, and intuitive web interface.
- Visualize student performance with charts, statistics, and interactive tables.
- Prioritize accessibility with a colorblind-safe palette.

## Technical Requirements
### Data Generator (Python)
- Generate 20 students with realistic names, genders, and form classes.
- Assign scores (0–100) with a realistic distribution.
- Generate separate JSON and CSV files per subject.
- **Denormalized Data**: Include metadata (student ID, name, gender, form class) directly in each subject file to eliminate frontend joining.
- Compute statistics and generate `metadata.json`.
- Use only Python standard library.

### Frontend (HTML/CSS/JS)
- **Landing page (`index.html`)**: A searchable global directory of student cards. Each card shows overall performance and subject badges. Clicking a badge navigates to the subject page.
- **Subject page (`subject.html`)**: Dynamic page loaded via `?subject=X`. Renders:
  - Statistics dashboard (Average, highest, passing rate).
  - Bar chart (Sorted by score, color-coded by performance band).
  - Doughnut chart (Grade distribution).
  - Student table (Sortable, searchable, color-coded rows).
- **Navigation**: Seamless back-to-home from subject pages.
- **Tech Stack**: Vanilla HTML/CSS/JS, Chart.js (CDN). No backend.
